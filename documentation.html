Documentation

The basic rules

You have a cascade levels of styles and classes.

  1. First the font-family basic rules.
  2. Followed of the basic reset styles which are general from all the projects or as it's called in codymer "themes". You can see the result in this basic reset page.
  3. After the general reset, you can the custom reset. Particular styles from each theme. This styles are preceded of the "cdmr--theme-(name)" class. You can see the result in this custom reset page.
  4. Then you have the general helpers. Classes you can use in any theme and works the same way, for example: "helper-hide", which allow you to hide the visibility of an element, without using display: none. You can see the list in the helpers page.
  5. After the general helpers, you have the custom helpers. Classes called the same way in all themes but working different, as you decide, in each one you work in. For example: "helper-bg-color-primary" can have one color in one theme and a different one in the other. You can see the list in the helpers page.
  6. You can add a layout system, based in CSS grid, if you need. The Layout folder is divided for the grids, the headers and the footers for each theme you need. You can go to in the layout grid page to see how it works.
  7. At this point comes the codymers, called as "cdmr-" in the SASS and CSS files. The most basic element in the system which can't or shouldn't be devided anymore, for example: "cdmr-btn".

    Here it's the full codymers list:

    • Breadcrumb
    • Btn
    • Btn iconized
    • Check
    • Form
    • Icon
    • Label
    • Link
    • List
    • Pagination
    • Radio
    • Switch
    • Table
    • Title
    • Txt
  8. With all the codymers you can add different types of containers. All of them starts with "cont-" the first one is the "cont-group-": this is a group of elements that grouped creates an element that works as a whole. For example: "cont-group-form-field".

    Here it's the full codymers containers type groups:

    • container group: form-field
  9. The text type of container or cont is the one called "set", which starts with "cont-set-". This is a type of container that contains elements of the same nature, for example a set of btn "cont-set-btn" or a set of groups like "cont-set-form-field".

    Here it's the full codymers containers type sets:

    • container set: btn
    • container set: form-field
    • container set: label
  10. And finally the container wrapper, called "cont-wrapper-" that contains codymers, groups, sets or even a layout. This can be used for full custom code, made entirely without any of the levels previously described but using BEM rules for the making.

    Here it's an example of a wrapper:

    • container wrapper: col-contents

The SASS architecture

To work with SASS you have the following folders and files architecture:

  1. abstracts/

    The folder for the functions, maps, mixins, variables that can be used in all themes. All grouped in a index.scss file at the root of the folder.

  2. base/

    The folder for the /helpers and the reset that can be used in all themes. All grouped in a index.scss file at the root of the folder.

  3. externals/

    : The folder for the styles you need to add for something external to your themes, an iframe for example. This folder is empty, feel free to remove it if you don't need to add anything like that.

  4. layout/

    The folder for the /footers, /grids and /headers.

  5. codymers/

    The folder for the /codymers, the basics elements in the system, like buttons, breadcrumbs, etc.

  6. containers/

    The folder for the /containers, is for the containers types: groups, sets and wrappers.

  7. vendors/

    If you need to add styles from other libraries this is the folder to organize them. The folder is empty because no vendors were needed for Codymer. Feel free to delete it if you don't need any vendor.

  8. themes/

    Inside is divided by folders, named as each project of theme you need. They contain folders like pages or the prints styles needed specifically for each theme. One of the most important file in this folder is setup, where you can custom the font-family rules, custom reset, custom helpers or layout grid. You can remove or add more if you need to.

The compilers

All the SASS architecture ends in the compilers. SCSS files that compiles a final .CSS for each theme. The compilation order is the next one:

  1. Core

    @use 'abstracts/index' as *; @use 'base/index' as *; @use 'codymers/index' as *; @use 'containers/index' as *;. These are the basic rules and helpers to setup your theme, each one has its own setup file.

  2. Setup

    @use 'themes/codymer/setup'; where font rules and mixin for each themes are loaded.

  3. Wrappers

    You can load general o specific wrappers for each theme if you want to.

This in a completed compiler example for the styles-page-codymer.css final file: 

// Sass Document

              // CORE
              // ===============================================
              @use 'abstracts/index' as *;
              @use 'base/index' as *;
              @use 'codymers/index' as *;
              @use 'containers/index' as *;
              // ===============================================
              
              // THEME SETUP
              // ================================================
              @use 'themes/codymer/setup';
              // ================================================
              
              // LAYOUT
              // ================================================
              // ================================================
              
              // VENDORS
              // ================================================
              // ================================================
              
              // WRAPPERS
              // ===============================================
              @include cont-wrapper-aside-structure();
              @include cont-wrapper-col-contents();
              // ===============================================

The setup file

This file, path in scss/themes/(themename)/_setup.scss is where you can set the basic rules of each theme.

The core

At the top of the file you have the Codymer's core by @use in order to use all functions, maps, mixins, variables, reset, helpers or grid. 

// CORE
              // ===============================================
              @use '../../abstracts/index' as *;
              @use '../../base/index' as *;
              @use '../../layout/grids/grid' as *;
              @use '../../codymers/index' as *;
              @use '../../containers/index' as *;
              // ===============================================

The font-family rules

After the core, you will setup the font-family basic orders. By the @include font-family() and #{$fonts-path} variable for locate the font's resources. For example Codymer uses Google Space Grotesk. 

// &--medium
            @include font-family("Google-Space-Grotesk--light",
              "#{$fonts-path}/google-space-grotesk/SpaceGrotesk-Light",
              300,
              normal,
              eot woff2 woff ttf svg);

            // &--regular
            @include font-family("Google-Space-Grotesk--regular",
              "#{$fonts-path}/google-space-grotesk/SpaceGrotesk-Regular",
              500,
              normal,
              eot woff2 woff ttf svg);

            // &--bold
            @include font-family("Google-Space-Grotesk--bold",
              "#{$fonts-path}/google-space-grotesk/SpaceGrotesk-Bold",
              700,
              normal,
              eot woff2 woff ttf svg);

The global reset

This is a global reset, where you can custom some of the basic styles, like the color or background of the body tag. This can be done by using the maps like: @include resets-global(); or map-fonts(), you cand find in scss/abstracts/maps/fonts and map-colors(), in scss/abstracts/maps/colors.

If you don't need to set some of the options, you can let them as null if you want. 

// RESET: global
            // ===============================================
            @include resets-global(

              // HTML FONT STYLE
              normal,

              // HTML FONT VARIANT
              normal,

              // HTML FONT WEIGHT
              500,

              // HTML FONT SIZE
              16px,

              // HTML LINE HEIGHT
              18px,

              // HTML FONT FAMILY
              map-fonts(Google, Space-Grotesk--300),

              // BODY FONT SIZE
              null,

              // BODY LINE HEIGHT
              null,

              // BODY FONT COLOR
              map-colors(purples, theme-codymer--600),

              // BODY LETTER SPACING
              null,

              // BODY BACKGROUND COLOR
              map-colors(pinks, theme-codymer--200),

            );
            // ===============================================

The custom reset

At this point this is a more specific reset for each theme. First of all the cdmr--theme-(name) in the body tag, and the other basic rules like the font-family you want to set or :selection behavior. 

// RESET: custom
            // ===============================================
            @include resets-custom(

              // THEME NAME
              codymer,

              // FONT FAMILY PRIMARY
              map-fonts(Google, Space-Grotesk--300),

              // FONT FAMILY SECONDARY
              map-fonts(Google, Space-Grotesk--500),

              // RESET TEXT SELECTION COLOR
              map-colors(blacks, theme--all),

              // RESET TEXT SELECTION BACKGROUND COLOR
              map-colors(yellows, theme-codymer--500),

              // RESET LINK SELECTION COLOR
              map-colors(whites, theme--all),

              // RESET LINK SELECTION BACKGROUND COLOR
              map-colors(purples, theme-codymer--600),

            );
            // ===============================================

The helpers

After the basic rules, comes the helpers. Some of them has the same styles in all themes. Like @helper-hides(); that allows you to hide elements, no matter the theme: 

// HELPER: hides
            // ===============================================
            @include helper-hides();
            // ===============================================

And others can be customize depending on the theme needs. For example @include helper-scope(example-01);, that allows you to set different font sizes with different behaviors in each theme you set. 

// HELPER: scope
              // ===============================================
              @include helper-scope(example-01);
              // ===============================================

The functions

All the functions are @forward in the functions.scss file.

_theming

It's used to set the theme-name in the body. You can see an example of this use in scss/base/_reset.scss: @mixin resets-custom();. 

#{functionBrandSet($resets-custom--brand)} {}

The maps

All the maps are @forward in the scss/abstracts/maps/maps.scss file.

_animation

You can set a collection of animations. You can see an example of this use in scss/abstracts/maps/_animations.scss. 

#{functionBrandSet($resets-custom--brand)} {}

.cdmr--theme-codymer tag/.class {}

_colors

A collection of colors where you can have all the themes palettes in one file. You can add or change them in the scss/abstracts/maps/_colors file. An example of using map-colors: 

border-color: map-colors(purples, theme-codymer--600);

The output: 

border-color: #1E1559;

_filters

Very useful to give colors to the SVG path icons. The whole filters collection is in the scss/abstracts/maps/_filters.scss. 

:before {
                      filter: map-filters(purples, theme-codymer--600);

The output:

_behavior

There is only one behavior mixin in the file @mixin behavior-in-ancestor() {}, that allows you to control the styles of any element when it's placed in different ancestor. 

    // &:where
                &:where( 
                    #{ $self }--secondary,
                    #{ $self }--quaternary
                  ) {
                  // .cdmr
                  .cdmr {
                    // &-icon
                    &-icon {
                      // &:before
                      &:before {
                        // theme: codymer
                        @if $theme == codymer {
                          filter: map-filters(blues, theme-codymer--50);
                        }
            
                        // theme: example-01
                        @if $theme == example-01 {
                          filter: map-filters(blacks, theme--all);
                        }
                      }
                    }
                  }
            
                  // behavior-in-ancestor
                  @include behavior-in-ancestor(helper-bg-color-primary-50) {
                    border-color: map-colors(whites, theme--all);
            
                    // .cdmr
                    .cdmr {
                      // &-icon
                      &-icon {
                        // &:before
                        &:before {
                          filter: map-filters(whites, theme--all);
                        }
                      }
                    }
                  }
            
                  // &:hover
                  &:hover {
                    // theme: codymer
                    @if $theme == codymer {
                      border-color: map-colors(purples, theme-codymer--50);
                    }
            
                    // theme: example-01
                    @if $theme == example-01 {
                      border-color: map-colors(blacks, theme-example-01--50);
                    }
            
                    // .cdmr
                    .cdmr {
                      // &-icon
                      &-icon {
                        // &:before
                        &:before {
                          // theme: codymer
                          @if $theme == codymer {
                            filter: map-filters(blacks, theme--all);
                          }
            
                          // theme: example-01
                          @if $theme == example-01 {
                            filter: map-filters(blacks, theme--all);
                          }
                        }
                      }
                    }
                  }
                }

_border

The @mixin border() {} Provides the border shorthand propieties.

First you set the border direction all, top, right, bottom, left. After the border-size, the border-style and finally the border-color. If you set as null one of the propieties, the style will not be processed in the final .CSS file. 

// CDMR: btn
              // ===============================================
              .cdmr-btn {
                @include border(all, .188rem, solid, null);
              }

The ouput: 

.cdmr-btn {
                border: 0.188rem solid;
              }

_hide

The @mixin hide-visually() {} hide elements without using display: none; and @mixin unhide-visually() {} does the opposite. 

// MIXIN HELPER: links
            // ===============================================
            @mixin helper-links() {

              // .helper-link
              .helper-link {

                // &-block
                &-block {
                  @include hide-visually;
                }

              }

            }

The ouput: 

.helper-link-block {
              clip: rect(0 0 0 0);
              clip-path: inset(50%);
              height: 1px;
              overflow: hidden;
              position: absolute;
              white-space: nowrap;
              width: 1px;
            }

_margin

Based in bourbon.io this @mixin margin() {} gives you the shorthand of the margin properties, depending on the direction all, top, right, bottom and left you use, for example: 

// MIXIN HELPER: alignments
          // ===============================================
          @mixin helper-alignments() {

            // .helper-algn
            .helper-algn {

              // &-ctr
              &-ctr {
                @include margin(null auto null auto);
              }

            }

          }

The ouput: 

.helper-algn-ctr {
            margin-right: auto;
            margin-left: auto;
          }

Another example: 

// MIXIN HELPER: positioning
            // ===============================================
            @mixin helper-positioning() {

              // .helper-pos
              .helper-pos {

                // &-ctr
                &-ctr {
                  @include margin(auto);
                }

              }

            }

The ouput: 

.helper-pos-ctr {
              margin: auto;
            }

_media-queries

Provided by Rafal Bromirski this mixin provides a useful set of media-queries.

Codymer uses @include min-screen() {} guided by Mobile First coding. This is an example: 

// CONT WRAPPER: col contents
            // ===============================================
            .cont-wrapper-col-contents {

              // &__col
              &__col {
                @include padding(1.563rem null 1.563rem null);
                @include size(100%, null);
                @include min-screen(1440px) {
                  @include padding(2.813rem null 2.813rem null);
                }

              }

            }

The ouput: 

.cont-wrapper-col-contents__col {
              padding-top: 1.563rem;
              padding-bottom: 1.563rem;
              width: 100%;
            }
            @media only screen and (min-width: 1440px) {
              .cont-wrapper-col-contents__col {
                padding-top: 2.813rem;
                padding-bottom: 2.813rem;
              }
            }

            }

          }

.cdmr-check__label {
            padding-left: 2rem;
          }

_size

The @mixin size() {} mixin set the with and height properties: 

// CDMR: switch
          // ===============================================
          .cdmr-switch {
            @include size(4rem, 2.5rem);
          }

The ouput: 

.cdmr-switch {
            height: 2.5rem;
            width: 4rem;
          }

If you set the @include size(); with one value, with and height both will have this one: 

// MIXIN HELPER: embedded
          // ===============================================
          @mixin helper-embedded() {

            // .helper-embed
            .helper-embed {

              // &-resp
              &-resp {

                // &__item
                &__item {
                  @include size(100%);
                }

              }

            }

          }

The ouput: 

.helper-embed-resp__item {
            height: 100%;
            width: 100%;
          }

_triangle

The @mixin triangle(size, color, direction) {} mixin helps you to make a CSS triangle where you need: 

// .class-name
          .class-name {

            // &:before
            &:before {
              content: "";
              @include triangle(.625rem, map-colors(purples, theme-codymer--600), up);
            }

          }

The ouput: 

.class-name:before {
            content: "";
            height: 0;
            width: 0;
            border-bottom: 0.3125rem solid #1E1559;
            border-left: 0.3125rem solid transparent;
            border-right: 0.3125rem solid transparent;
          }

SASS coding

A couple of basic rules to code your SASS in Codymer:

  • Add the @use './abstracts/index' as *; to the files in order to access all the basic tools, like functions, maps, mixins, variables. This is Codymer's core. 
    // CORE
                  // ===============================================
                  @use './abstracts/index' as *;
                  // ===============================================
  • Try to keep a properties order, I suggest 9 elements css rule order
  • Remember all is about BEM.
  • The $self: &; is really useful. You can read more information in this CSS Trick's article .
  • Use modifier: &--primary, &--secondary, &--... and so on for the different codymers or containers types.
  • Use modifier: 25, 50, ... or the number scale you decide, if you need to sizes, weights, shapes or anything you need to scale in a list. Follow the Mobile First rules for this, that means you can scale from the smaller resolution to the highest you need. 
    .helper-font-scope-10 {

                  line-height:1rem
